# Contributing to Stackby API Unofficial Docs

Thank you for considering contributing! This documentation helps developers integrate with Stackby.

## How to Contribute

### Reporting Issues

If you discover a new API behavior, bug, or undocumented feature:

1. Check if it's already documented in [KNOWN_ISSUES.md](KNOWN_ISSUES.md)
2. Open a GitHub issue with:
   - Clear description of the behavior
   - Reproduction steps (curl command preferred)
   - Expected vs actual behavior
   - Any error messages

### Adding Documentation

1. Fork the repository
2. Create a feature branch
3. Add your documentation/examples
4. Submit a pull request

### Code Examples

When adding code examples:

- Use clear, readable code
- Include comments explaining key points
- Test your code before submitting
- Follow existing formatting conventions

## Guidelines

- Keep the tone constructive and helpful
- Focus on facts and technical accuracy
- Avoid complaints; document solutions
- Include working code whenever possible

## Questions?

Open an issue and we'll help!
